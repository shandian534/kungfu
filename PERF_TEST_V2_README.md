# 易筋经性能测试 V2 使用说明

## 测试逻辑

V2 版本按照**文件级别**的批处理逻辑：

### 1. 顺序写入 (Sequential)
- **逻辑**：每次处理 **1个文件**，完成后处理下一个
- **适用场景**：测试单文件处理的基准性能
- **特点**：串行处理，资源占用低

```
文件1 → [解析+写入] → 完成 → 文件2 → [解析+写入] → 完成 → ...
```

### 2. 批量写入 (Batch)
- **逻辑**：每次同时处理 **N个文件**（如10个、100个）
- **适用场景**：测试多文件聚合处理的性能
- **特点**：一次性解析N个文件，然后批量写入

```
[文件1-10] → [全部解析] → [全部写入] → [文件11-20] → ...
```

### 3. 并行写入 (Parallel)
- **逻辑**：**N个进程**同时工作，每个进程处理1个文件
- **适用场景**：测试多进程并发写入的性能
- **特点**：多个 apprentice 同时连接 master

```
进程1: 文件1 → [解析+写入]
进程2: 文件2 → [解析+写入]
进程3: 文件3 → [解析+写入]
进程4: 文件4 → [解析+写入]
进程5: 文件5 → [解析+写入]
→ 全部同时进行
```

## 使用方法

### 1. 启动 Master

```bash
export KF_HOME=/Users/shandian/kungfu
kfc run -c system -g master -n master &
```

### 2. 运行测试

```bash
# 快速验证 (10文件，顺序)
python3.9 perf_test_v2.py --scenario 0

# 批量写入 (1000文件，每次10个)
python3.9 perf_test_v2.py --scenario 4

# 并行写入 (1000文件，5个进程)
python3.9 perf_test_v2.py --scenario 10

# 大规模测试 (5000文件，10个进程)
python3.9 perf_test_v2.py --scenario 15
```

### 3. 清理临时文件

```bash
./cleanup_test.sh
```

## 测试建议

### 测试顺序

1. **先运行快速验证**（场景0）：确保环境正常
2. **再运行小规模测试**（场景4-5）：验证批处理逻辑
3. **最后运行大规模测试**（场景8-9、14-15）：获取完整性能数据

### 性能对比

通过对比不同场景可以分析：
- **顺序 vs 批量**：批处理的性能优势
- **批量10 vs 批量100**：最佳批量大小
- **并行5 vs 并行10**：最佳并行数
- **解析 vs 写入**：性能瓶颈在哪

## 注意事项

1. **每次只能运行一个场景**（由于框架限制）
2. **大规模测试前建议清理 journal**：`./cleanup_test.sh`
3. **确保 master 正常运行**：检查进程状态
4. **超时错误可忽略**：不影响测试结果，框架的已知问题
