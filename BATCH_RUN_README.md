# 批量运行所有测试场景

## 两种批量运行方式

### 方式 1：Shell 脚本（推荐）

```bash
./run_all_scenarios_v2.sh
```

**优点：**
- 简单直接
- 兼容性好
- 输出实时显示

### 方式 2：Python 脚本

```bash
python3.9 run_all_scenarios_v2.py
```

**优点：**
- 更灵活
- 便于修改和扩展
- 可以添加更多统计功能

## 功能特点

### 1. 进度显示
- 百分比进度：`[0%]`, `[5%]`, `[10%]`...
- 当前场景：`场景 3/19`
- 场景名称：`S3_顺序_3000文件`

### 2. 时间统计
- **单场景耗时**：每个场景完成后显示
- **总耗时**：所有场景完成后显示
- **格式化显示**：
  - 秒：`15s`
  - 分秒：`2m 35s`
  - 时分秒：`1h 25m 30s`

### 3. 结果汇总
- **成功/失败统计**
- **详细结果表格**（索引 | 名称 | 状态 | 耗时）
- **按类型分组**（顺序 | 批量 | 并行）

### 4. 自动等待
- 每个场景后等待 3 秒
- 确保资源正确释放

## 使用前检查

### 1. 确保 Master 运行

```bash
pgrep -f "kfc run.*master"
```

### 2. 如果没有 Master，启动它

```bash
export KF_HOME=/Users/shandian/kungfu
kfc run -c system -g master -n master &
```

### 3. 清理旧数据（可选）

```bash
./cleanup_test.sh
```

---

## 注意事项

1. **总耗时较长**
   - 20个场景预计需要 3-5 小时
   - 建议在后台运行或使用 screen/tmux

2. **中途停止**
   - 按 `Ctrl+C` 停止
   - 已完成的场景结果会保留

3. **失败重试**
   - 如果某个场景失败，继续运行下一个
   - 最后可以看到哪些场景失败了
   - 可以单独重跑失败的场景

4. **磁盘空间**
   - 确保有足够空间（建议 > 10GB）
   - Journal 文件会占用大量空间

## 后台运行示例

### 使用 nohup

```bash
nohup ./run_all_scenarios_v2.sh > test_run.log 2>&1 &
```

查看进度：
```bash
tail -f test_run.log
```

### 使用 screen

```bash
screen -S perf_test
./run_all_scenarios_v2.sh
# 按 Ctrl+A 然后 D 分离
```

重新连接：
```bash
screen -r perf_test
```

### 使用 tmux

```bash
tmux new-session -d -s perf_test './run_all_scenarios_v2.sh'
tmux attach-session -t perf_test
```
